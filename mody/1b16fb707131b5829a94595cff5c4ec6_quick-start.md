# Quick-Start Workflow

When the user runs `/quick-start`, perform the following:

1. **Identify the active project** from open files, running terminals, or workspace context.
2. **Read key project files**: `package.json`, `README.md`, `.env.example`, config files, `docker-compose.yml`, and any project standards docs.
3. **Explore** `src/` directory structure (app routes, components, lib, hooks, types).
4. **Produce a concise summary** covering:
   - Tech stack & versions
   - How to run locally (prerequisites, env setup, Docker, dev server)
   - Project structure overview (routes, components, API, DB)
   - Key scripts & commands
   - Project standards / conventions
5. **Output** the summary directly to the user as a formatted message.
